### 一、睡眠的理发师问题
#### 原题复刻
```c
#define CHAIRS 5    /* # chairs for waiting customers */
semaphore customers = 0;    /* # of customers waiting for service */
semaphore barbers = 0;     /* # of barbers waiting for customers */
semaphore mutex = 1;       /* for mutual exclusion */
int waiting = 0;           /* customers are waiting (not being cut) */

void barber(void)
{
    while (TRUE) {
        P(____1____);    /* go to sleep if # of customers is 0 */
        P(mutex);        /* acquire access to 'waiting' */
        waiting = waiting - 1; /* decrement count of waiting customers */
        V(barbers);      /* one barber is now ready to cut hair */
        V(mutex);        /* release 'waiting' */
        cut_hair();      /* cut hair (outside critical region) */
    }
}

void customer(void)
{
    P(____2____);        /* enter critical region */
    if (waiting < CHAIRS) {  /* if there are no free chairs, leave */
        waiting = waiting + 1;/* increment count of waiting customers */
        V(____3____);    /* wake up barber if necessary */
        V(mutex);        /* release access to 'waiting' */
        P(____4____);    /* go to sleep if # of free barbers is 0 */
        get_haircut();   /* be seated and be serviced */
    } else {
        V(____5____);    /* shop is full; do not wait */
    }
}
```
#### 填空答案
1. `customers`  2. `mutex`  3. `customers`  4. `barbers`  5. `mutex`
#### 详细解析
1. **空1（`customers`）**：理发师没有顾客时需要休眠，`P(customers)`会阻塞理发师；顾客到店会`V(customers)`唤醒理发师，实现睡眠-唤醒同步。
2. **空2（`mutex`）**：变量`waiting`是多进程共享的临界资源，进入修改`waiting`的临界区前，必须`P(mutex)`获取互斥锁。
3. **空3（`customers`）**：顾客成功坐到等候椅后，通过`V(customers)`给等待顾客的理发师发信号，唤醒理发师工作。
4. **空4（`barbers`）**：理发师准备好理发后会执行`V(barbers)`，顾客需要`P(barbers)`阻塞等待理发师就绪，才能开始理发。
5. **空5（`mutex`）**：店内椅子已满、顾客直接离开时，必须释放之前获取的互斥锁`mutex`，否则会造成死锁。

### 二、仓库存放两种产品问题
#### 原题复刻
有一个仓库，可以存放A和B两种产品，仓库的存储空间足够大，要求：
(1) 一次只能存入一种产品（A或B）；
(2) $-N \leq (\text{A产品数量}-\text{B产品数量}) \leq M$。
试用PV操作描述产品A与产品B的入库过程。
```c
semaphore ticket_a=M, ticket_b=____1____;
put_a(){
    P(____2____);
    put a into store;
    V(____3____);
}
put_b(){
    P(____4____);
    put b into store;
    V(____5____);
}
```
#### 填空答案
1. `N`  2. `ticket_a`  3. `ticket_b`  4. `ticket_b`  5. `ticket_a`
#### 详细解析
先拆解数量约束不等式：
- $\text{A}-\text{B} \leq M$：每新增1个A，差值会+1，用初值为$M$的`ticket_a`控制上限，放A前`P(ticket_a)`申请额度；
- $\text{B}-\text{A} \leq N$：每新增1个B，差值会+1，用初值为$N$的`ticket_b`控制上限，放B前`P(ticket_b)`申请额度；
- 放A后$\text{A}$变多，B的可新增额度增加，因此`V(ticket_b)`；放B后$\text{B}$变多，A的可新增额度增加，因此`V(ticket_a)`；
- “一次只能存一种”的互斥要求，可在`put into store`内部通过临界区实现，同步信号量完成差值约束控制。

1. **空1（`N`）**：对应$\text{B}-\text{A} \leq N$的约束，初始化`ticket_b=N`。
2. **空2（`ticket_a`）**：存放A前申请A的增量额度，避免$\text{A}-\text{B}$超过上限$M$。
3. **空3（`ticket_b`）**：A入库后，B的可存放上限放宽，释放B的额度。
4. **空4（`ticket_b`）**：存放B前申请B的增量额度，避免$\text{B}-\text{A}$超过上限$N$。
5. **空5（`ticket_a`）**：B入库后，A的可存放上限放宽，释放A的额度。

### 三、吸烟者问题
#### 原题复刻
三个吸烟者在一间房间，还有一个香烟供应者。制造香烟需要烟草、纸、火柴三样物品：
- 吸烟者P2自有烟草，缺纸+火柴；
- 吸烟者P3自有纸，缺烟草+火柴；
- 吸烟者P4自有火柴，缺烟草+纸；
供应者随机往桌上放两样物品，唤醒匹配的吸烟者；吸烟者抽完烟后唤醒供应者放下一批物品。
```c
semaphore offer1=0;//烟草+纸组合
semaphore offer2=0;//烟草+火柴组合
semaphore offer3=0;//纸+火柴组合
semaphore finish=____1____;//表示抽烟是否完成
process P1(){//供应者
    int i=0;
    while (1)
    {
        if(i==0)
            V(offer1);//放烟草和纸
        else if(i==1)
            V(offer2);//放烟草和火柴
        else if(i==2)
            V(offer3);//放纸和火柴
        P(____2____);
        i=(i+1)%3;
    }
}
process P2(){//自有烟草
    while(1){
        P(____3____);
        //拿纸和火柴，卷烟抽烟
        V(finish);
    }
}
process P3(){//自有纸
    while(1){
        P(____4____);
        //拿烟草和火柴，卷烟抽烟
        V(finish);
    }
}
process P4(){//自有火柴
    while(1){
        P(____5____);
        //拿烟草和纸，卷烟抽烟
        V(finish);
    }
}
```
#### 填空答案
1. `0`  2. `finish`  3. `offer3`  4. `offer2`  5. `offer1`
#### 详细解析
1. **空1（`1`）**：finish 的作用是让老板 “等顾客抽完再做下一份”。最开始还没有任何订单完成，所以初始值为0。
2. **空2（`finish`）**：供应者放出物品后阻塞，等待吸烟者抽完烟、`V(finish)`唤醒，才能继续放下一批。
3. **空3（`offer3`）**：P2自带烟草，只有桌上出现纸+火柴（`offer3`）时，才能凑齐材料抽烟，因此`P(offer3)`。
4. **空4（`offer2`）**：P3自带纸，只有桌上出现烟草+火柴（`offer2`）时，才能凑齐材料抽烟，因此`P(offer2)`。
5. **空5（`offer1`）**：P4自带火柴，只有桌上出现烟草+纸（`offer1`）时，才能凑齐材料抽烟，因此`P(offer1)`。

### 四、天南大路路问题
#### 原题复刻
天津大学与南开大学间的小路：每个方向同一时间仅允许一辆车通行；安全岛M最多停留两辆车（双向各一辆错车）。
信号量定义：
- `T=1`：天大路口资源互斥
- `S=1`：南开路口资源互斥
- `L=1`：天大→安全岛M路段互斥
- `K=1`：南开→安全岛M路段互斥
```c
semaphore T=1;
semaphore S=1;
semaphore L=1;
semaphore K=1;
process_from_tju_to_nku(){//天大→南开方向
    P(____1____);
    P(____2____);
    通过L路段;
    进入安全岛M;
    V(____3____);
    P(____4____);
    通过K路段;
    V(____5____);
    V(T);
}
process_from_nku_to_tju(){//南开→天大方向逻辑相反，略}
```
#### 填空答案
1. `T`  2. `L`  3. `L`  4. `K`  5. `K`
#### 详细解析
天大→南开的车辆通行流程：
1. **空1（`T`）**：先占用天大入口路口`T`，锁定入口，保证该方向一次仅一辆车进入小路。
2. **空2（`L`）**：再占用天大到安全岛的`L`路段，锁定这段路的通行权。
3. **空3（`L`）**：车辆驶入安全岛M后，`L`路段空闲，释放`L`让后续车辆使用。
4. **空4（`K`）**：接下来要走安全岛到南开的`K`路段，提前申请`K`的通行权。
5. **空5（`K`）**：车辆通过`K`路段驶出后，释放`K`路段资源，最后释放天大路口`T`。

### 五、游泳馆保洁问题
#### 原题复刻
游泳馆规则：
1. 同时在场游泳人数上限为$N$，满员时新游客场外排队；
2. 保洁打扫时暂停新游客入场；
3. 保洁需等场内所有游客离场后才能开始打扫；
4. 打扫完成后重新开放入场。
```c
semaphore mutex=1;  //互斥访问count变量
semaphore limit=N;  //同时入馆人数上限控制
semaphore pool=1;   //场馆整体互斥（保洁独占）
semaphore close=1;  //入场开关控制
int count=0;        //场内当前游泳人数

swimmer(){
    P(____1____);
    P(mutex);
    count++;
    if(count==1) P(____2____);
    V(mutex);
    V(____3____);
    P(____4____);
    // 游泳
    P(mutex);
    count--;
    if(count==0) V(____5____);
    V(mutex);
    V(limit);
}

cleaner()
{
    P(close);
    P(pool);
    V(close);
    // 打扫卫生
    V(pool);
}
```
#### 填空答案
1. `close`  2. `pool`  3. `close`  4. `limit`  5. `pool`
1. **横线1 P(close)**：先拿入场开关。保洁执行`P(close)`后，所有新游客都会阻塞在这里，实现“打扫时暂停入场”。
2. **横线2 P(pool)**：第一个进场游客锁住场馆pool，保洁无法获取pool，必须等所有人离场。
3. **横线3 V(close)**：拿到入场资格后，释放close，允许其他游客依次进场排队。
4. **横线4 P(limit)**：占用1个人数名额，控制场内总人数≤N。
5. **横线5 V(pool)**：最后一名游客离场，释放场馆pool，保洁可以拿到pool开始打扫。

## 第1题
### 题干
【单选题】(2分)
有6个CPU密集型批处理作业A、B、C、D、E和F，几乎同时被提交。预计运行时间分别为12，6，2，4，8和2分钟。忽略进程切换的开销。设进程A-F的优先级分别为4，6，3，2，5和1，其中1为最高优先级。则采用优先级调度算法，平均进程周转时间为（）
A. 16.33分钟
B. 14.33分钟
C. 14分钟
D. 23.33分钟
### 解答
优先级数值越小优先级越高，所以调度执行顺序从先到后为：F(优先级1，运行2min) → D(优先级2，运行4min) → C(优先级3，运行2min) → A(优先级4，运行12min) → E(优先级5，运行8min) → B(优先级6，运行6min)
逐个计算周转时间（周转时间=作业完成时刻-到达时刻，作业几乎同时到达，到达时刻为0，周转时间等于累加完成时间）：
- F：完成时刻=2，周转时间=2
- D：完成时刻=2+4=6，周转时间=6
- C：完成时刻=6+2=8，周转时间=8
- A：完成时刻=8+12=20，周转时间=20
- E：完成时刻=20+8=28，周转时间=28
- B：完成时刻=28+6=34，周转时间=34
周转时间总和：$2+6+8+20+28+34=98$
平均周转时间：$98\div6\approx16.33$分钟
### 参考答案：A

【单选题】(2分)
系统中有3个不同的临界资源R1、R2和R3，被4个进程P1、P2、P3和P4共享。各进程对资源的需求为：P1申请R1和R2，P2申请R2和R3，P3申请R1和R3，P4申请R2。若系统出现死锁，则处于死锁状态的进程数至少是（）
A. 1
B. 2
C. 3
D. 4
### 解答

P4只需要R2，只要R2空闲就可以获取资源执行完成、释放资源，不会卷入死锁。
因此死锁进程数至少为3。
### 参考答案：C
